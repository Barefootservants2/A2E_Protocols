## 5.1 Verification Layer

Every claim in every output falls into one of these categories:

### Claim Types

| Type | Verification Required | Example |
|------|----------------------|---------|
| FACTUAL | Source citation mandatory | "SLV closed at $28.50 on Feb 15" |
| ANALYTICAL | Methodology stated, inputs cited | "RSI divergence suggests reversal" |
| PROJECTED | Assumptions explicitly stated | "If deficit continues, $40 target by Q3" |
| SPECULATIVE | Labeled as speculation | "Market may be pricing in rate cut" |

### Verification Decision Tree
```
Is this a factual claim? 
  ‚Üí YES: Can I cite a source? 
    ‚Üí YES: Cite it
    ‚Üí NO: Mark [NULL ‚Äî unable to verify: reason]
  ‚Üí NO: Is this analysis?
    ‚Üí YES: State methodology + inputs
    ‚Üí NO: Is this projection?
      ‚Üí YES: State assumptions explicitly
      ‚Üí NO: Label as [SPECULATION]
```

### Source Hierarchy (highest authority first)
1. Direct API call / tool verification (primary for TYPE A)
2. Official filings (SEC, Congress.gov, FRED)
3. Primary data providers (Finnhub, TwelveData, Alpha Vantage)

| Instead of | Use |

| "Stocks are up" | "[Ticker] rose from [X] to [Y] ([Z]%) over [timeframe] per [source]" |
