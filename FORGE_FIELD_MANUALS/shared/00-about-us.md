We built FORGE because we got tired of that.

ANVIL scores the framework behind the answer — not the answer itself. ASSAY evaluates whether the methodology would survive scrutiny from someone who actually works in the field. AUTOPSY detects the difference between substance and performance — the engagement bait, the false confidence, the analysis that sounds brilliant but collapses the moment you ask "prove it."

These tools exist because we lost money trusting outputs we didn't verify. $4,500 in a single month. That loss became IRONCLAD, our risk protocol. And IRONCLAD's first rule is simple: data leads, Principal decides. AI informs. Humans own the consequence.

Every prompt in this series was tested in production — not in a sandbox, not in a classroom, but against live markets, real contracts, and actual stakes. The ones that failed were cut. The ones that survived are here, scored, cited, and shown with their results so you can verify what we claim.

## The Uriel Covenant AI Collective

The system behind this book is not a single AI. It is a production-grade multi-agent orchestration layer running five large language models in parallel: GPT-4.1 (URIEL), Gemini (HANIEL), DeepSeek (RAZIEL), Grok (COLOSSUS), and Perplexity (SARIEL). Each agent processes the same query independently. Their outputs are synthesized through a confidence cascade — a nine-gate scoring system that requires consensus before any recommendation advances to action.

This architecture exists for one reason: no single model should be trusted alone. When three out of five agents agree on a finding, confidence increases. When they disagree, the system flags the divergence and forces deeper analysis. When RAZIEL — our dedicated counter-thesis agent — identifies a flaw in the consensus, that flaw gets surfaced, not buried.

The Collective was not purchased. It was built over eighteen months of iterative development on n8n Cloud, GitHub, and Claude Code. It runs overnight watch cycles, market scans, and analytical pipelines without human intervention. But it does not make decisions. Decisions belong to the Principal.

## How to Use This Book

Each volume in the FORGE Field Manual series follows the same structure:

**Foundation Prompts** are starting points. They establish context, define the analytical frame, and produce first-pass outputs. Use them when you need to orient quickly in a domain you understand but haven't yet asked AI to help with.

**Advanced Chains** are multi-step sequences. Each prompt builds on the output of the previous one, creating layered analysis that a single prompt cannot achieve. These are where the real value lives — and where most prompt libraries stop short.

**ANVIL Scoring** is the framework chapter. It teaches you to evaluate what AI gives you back — not by gut feel, but by structured criteria specific to the domain. Every scored example in this book uses the ANVIL rubric so you can see exactly how we graded the output.

**ASSAY Evaluation** goes deeper. Where ANVIL scores the framework, ASSAY evaluates the methodology. Does the analysis use the right approach? Would an expert in the field accept this reasoning? ASSAY is the peer review layer.

**AUTOPSY Red Flags** is the detection chapter. It catalogs the patterns we've identified where AI produces confident-sounding garbage — the specific failure modes, the linguistic tells, the structural patterns that indicate an output should not be trusted. This chapter alone may save you more than the price of the book.

**Real Examples** show the system in action. Actual prompts, actual outputs, actual scores, actual commentary on what worked and what didn't. No cherry-picking. The failures are included because they teach more than the successes.

## A Note on Originality

Every framework in this series has a lineage. We drew from established methodologies — Porter's competitive analysis, Boyd's OODA loop, Munger's mental models, Deming's quality systems — and we cite those sources clearly. What we built on top of them is original: the scoring rubrics, the confidence cascades, the red-flag catalogs, the multi-agent verification architecture.

Where we adapted existing work, we say so and explain what we changed and why. Where we built from scratch, we show our reasoning. Where we were inspired by another practitioner's insight, we credit them by name.

This is not a prompt pack dressed up as a book. This is a practitioner's manual built by people who use these tools every day with real stakes on the line. The prompts are the delivery mechanism. The methodology is the product.
